# Security Policy

## Reporting Vulnerabilities
If you discover a security vulnerability within this project, please report it by opening an issue on GitHub. Alternatively, you can email us directly at security@example.com. We will confirm receipt of your report and provide further instructions.

## Supported Versions
We maintain support for the following versions of this project:
- Current version: v1.x (active support)
- Previous major version: v0.x (limited support)
  
Older versions may be vulnerable, and we encourage you to upgrade to the latest version.

## Security Best Practices
To maintain the security of your application and our library, we recommend the following best practices:
- Always use the latest version of this library.
- Regularly review and update your dependencies.
- Scan your application for vulnerabilities using tools like [OWASP Dependency-Check](https://owasp.org/www-project-dependency-check/) or [Snyk](https://snyk.io/).

## Contact Information
For any questions regarding the security policy, please reach out to us via email at security@example.com or through our GitHub page.
